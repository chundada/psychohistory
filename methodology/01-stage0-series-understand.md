# Stage 0：系列理解（Series Understanding）

> **目标**：在投入深度提取之前，先全面理解整个 YouTube 系列的结构、主题和核心论点。
> **输入**：YouTube 播放列表 URL 或剧集列表
> **输出**：`SERIES_OVERVIEW.md`（使用 `templates/SERIES_OVERVIEW.md.template`）

---

## 一、为什么需要 Stage 0

书籍蒸馏中，"阅读全书"是隐含的 Stage 0。但对于视频系列，情况更复杂：

- 视频长度从 10 分钟到 3 小时不等
- 每集是独立的，但属于更大的叙事弧
- 作者可能在系列中期修正自己的观点
- 有些剧集是 filler（填充内容），不值得深度提取

Stage 0 让你在全速投入之前先"看清地图"。

---

## 二、执行步骤

### Step 1：列出所有剧集

创建一个完整的剧集清单，包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| EP# | 剧集编号（在系列中的顺序） | GT#01 |
| 标题 | 视频标题（原始语言） | The Clash of Civilizations? |
| 时长 | HH:MM:SS | 2:15:30 |
| 发布日期 | YYYY-MM-DD | 2025-03-14 |
| 视频链接 | YouTube URL | https://youtube.com/watch?v=... |
| 初步印象 | 看完标题和简介后的第一判断 | 核心论点集，高优先级 |

### Step 2：分组为主题弧（Thematic Arcs）

将剧集按主题分组。典型的 YouTube 系列会有 3-5 个主题弧。

**分组原则**：
- 同一弧内的剧集共享同一个核心问题
- 弧之间存在逻辑递进关系（而非并列关系）
- 每个弧至少 3 集（少于 3 集的弧合并到相邻弧）

**示例**（假设系列为《Global Trends》）：
| 主题弧 | 剧集范围 | 主题 | 估计价值 |
|--------|---------|------|---------|
| Arc 1 | GT#01-GT#10 | 历史规律与文明基础 | 高（框架密集型） |
| Arc 2 | GT#11-GT#20 | 地缘政治与国际关系 | 高（案例密集型） |
| Arc 3 | GT#21-GT#25 | 技术革命与 AI | 中高（预测密集型） |
| Arc 4 | GT#26-GT#29 | 未来展望与政策建议 | 中（应用密集型） |

### Step 3：识别作者的核心论点与方法论

针对整个系列，回答以下问题：

1. **核心论点**：作者试图证明什么？用一句话概括整个系列的中心思想。
2. **方法论**：作者用什么方法得出论点？（历史比较法？数据驱动？案例研究？思想实验？）
3. **独特视角**：作者与其他人在这个主题上的不同之处在哪里？
4. **知识前提**：观众需要什么背景知识才能理解？这决定了技能的前提条件。

### Step 4：识别关键局限性

诚实评估系列的局限性：

1. **认知偏见**：作者是否有明显的意识形态倾向？
2. **数据可靠性**：引用来源的可信度如何？
3. **适用范围**：作者的论点在什么条件下成立/不成立？
4. **时效性**：哪些分析会随时间推移而失效？

### Step 5：标记"技能可用单元"

对于每集，初步判断其中包含的可提取技能数量：

| 标记 | 含义 | 处理策略 |
|------|------|---------|
| ✅ 高价值 | 包含 3+ 可提取技能 | 进入 Stage 1 完整提取 |
| ⚠️ 中等 | 包含 1-2 个框架 | 轻量提取即可 |
| ❌ 低价值 | 概述、回顾、嘉宾闲聊 | 仅做笔记，不提取 |
| 📦 只有背景 | 历史背景介绍，无框架 | 仅存档用于上下文 |

## 三、输出模板使用说明

使用 `templates/SERIES_OVERVIEW.md.template` 生成 `SERIES_OVERVIEW.md`。

### 必填字段（Quality Gate QG1）

```
series_name:          必须
total_episodes:       必须
total_duration:       必须
thematic_arcs[]:      至少 1 个
  arc_name:           必须
  episodes:           必须
  theme:              必须
core_thesis:          必须（一句话）
interpretive_framework: 必须（2-5 句）
critical_limitations: 至少 2 条
```

### 后续查询

完成 SERIES_OVERVIEW.md 后，回答以下问题以确定 Stage 1 的范围：

1. 这个系列值得完整蒸馏吗？如果否，停止。
2. 哪些主题弧优先提取？（优先级排序）
3. 每集的初步技能标记是什么？（✅/⚠️/❌/📦）
4. 是否有特定的提取器更适合这个系列？（例如，历史类系列多用 Case Study 提取器）

## 四、检查清单

- [ ] 所有剧集已列出（标题 + 时长 + 链接）
- [ ] 已分组为 3-5 个主题弧
- [ ] 核心论点已用一句话概括
- [ ] 方法论已识别
- [ ] 至少 2 个关键局限性已记录
- [ ] 每集已标记技能价值
- [ ] SERIES_OVERVIEW.md 已保存到工作区根目录

---

## 五、常见陷阱

| 陷阱 | 后果 | 避免方法 |
|------|------|---------|
| 跳过 Stage 0 直接提取 | 提取到 filler 内容，浪费资源 | 强制先完成 SERIES_OVERVIEW.md |
| 主题弧划分过细 | 管理复杂，弧间重叠 | 每弧至少 3 集 |
| 忽视局限性 | 技能适用范围不明确 | 至少写 2 条局限性 |
| 高估低价值剧集 | 投入产出比低 | 使用"预览速览法"：只看标题+简介+章节标记 |
