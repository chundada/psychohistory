# Stage 2：RIA+ 技能构建（RIA+ Construction）

> **目标**：从 Stage 1 的候选方法论中，直接构造可用的 SKILL.md
> **输入**：`candidates/` 目录中的候选文件
> **输出**：`skills/` 目录中的 SKILL.md 文件

---

## 一、RIA+ 格式（简化版）

> **v4.1 改进**：RIA++（6 字段）→ RIA+（4 字段）。
> 原 A1（作者应用）与 R（原文引用）高度重复，现合并。

```
┌──────────────────────────────────────┐
│              SKILL.md                  │
├──────────────────────────────────────┤
│  R ─ Reading（阅读原文）               │
│     视频转录的直接引用 ≤150 字          │
├──────────────────────────────────────┤
│  I ─ Interpretation（方法论解读）       │
│     方法论骨架，5-15 行                │
├──────────────────────────────────────┤
│  A ─ Application（应用与执行）          │
│     何时用 + 怎么用 + 作者案例         │
│    (原 A1+A2+E 合并为一个连续章节)     │
├──────────────────────────────────────┤
│  B ─ Boundary（边界条件）               │
│     何时不应该使用这个技能              │
└──────────────────────────────────────┘
```

---

## 二、内嵌验证（取代独立的四重验证阶段）

**不再单独运行 Stage 1.5。** 在构造每个 SKILL.md 时自然判断：

```
在写每个 SKILL.md 时回答三个问题：
① 能引用到原文吗？      → 有可追溯的 [source]
② 能提炼可操作步骤吗？  → 告诉别人"怎么做"
③ 知道何时不该用吗？   → 不是万能公式
```

如果以上全部是"是" → 该候选成熟，写出 RIA+
如果任何一个是"否" → 该候选不成熟，直接跳过，不花时间构造

---

## 三、每个字段的详细指南

### R：Reading（阅读原文）

**规则**：
- 直接引用视频转录，≤150 字
- 标注时间戳 `[HH:MM:SS - HH:MM:SS]`
- 转录不完整时标注 ⚠️

**格式**：
```markdown
## R — Reading（阅读原文）

> [12:34 - 15:20]
> "..."
>
> — EP03: <标题>
```

### I：Interpretation（方法论解读）

**规则**：
- 用自己的话提炼方法论骨架
- 5-15 行（约 150-400 字）
- 聚焦"这个技能的结构是什么"而非"这个技能说了什么"

### A：Application（应用与执行）

**把原来拆开的 A1(作者案例)+A2(触发条件)+E(执行步骤) 合并为一个完整章节。** 这样更自然：读者不需要在不同章节之间跳转。

建议结构（不必严格三段式，可以是流淌的叙述）：

```
1. **触发条件**：当___时应使用这个技能
2. **作者案例**：作者在视频中用这个框架分析了___（含时间戳）
3. **执行步骤**：
   步骤1. ___
   步骤2. ___
   步骤3. ___
```

### B：Boundary（边界条件）

**规则**：
- 至少 2 个不适用场景
- 包含"在什么条件下这个分析失效"

---

## 四、输出检查清单

- [ ] R：有直接引用 + 时间戳
- [ ] I：方法论清晰，5-15 行
- [ ] A：包含触发条件 + 执行步骤
- [ ] B：至少 2 个不适用场景
- [ ] 整个文件通过内嵌验证（三问皆"是"）
