# Stage 2：RIA++ 技能构建（RIA++ Construction）

> **目标**：将验证通过的候选技能转化为完整的 RIA++ 格式
> **输入**：`verified.md` 中通过四重验证的候选技能
> **输出**：`skills/` 目录中的 SKILL.md 文件

---

## 一、RIA++ 格式概览

RIA++ 在经典 RIA（Reading-Interpretation-Application）基础上扩展为六字段结构：

```
┌──────────────────────────────────────┐
│              SKILL.md                  │
├──────────────────────────────────────┤
│  R ─ Reading（阅读原文）               │
│     视频转录的直接引用 ≤150 字          │
├──────────────────────────────────────┤
│  I ─ Interpretation（解读）            │
│     方法论骨架，用自己的话 5-15 行       │
├──────────────────────────────────────┤
│ A1 ─ Application by Author（作者应用）  │
│     作者如何在实际分析中应用这个技能      │
├──────────────────────────────────────┤
│ A2 ─ Application Trigger（你的触发条件）│
│     你应该在什么情况下使用这个技能        │
├──────────────────────────────────────┤
│  E ─ Execution（执行步骤）              │
│     可操作的具体步骤                    │
├──────────────────────────────────────┤
│  B ─ Boundary（边界条件）               │
│     何时不应该使用这个技能               │
└──────────────────────────────────────┘
```

---

## 二、每个字段的详细指南

### R：Reading（阅读原文）

**规则**：
- 必须是视频转录中的**直接引用**，不可改写
- 引用长度 ≤150 字
- 必须标注时间戳：`[HH:MM:SS - HH:MM:SS]`
- 如果转录不完整，使用"近似引用"并标注 ⚠️

**格式**：
```markdown
## R — Reading（阅读原文）

> [12:34 - 15:20]
> "文明衰落的核心机制并非外部入侵，而是内部社会结构的弹性丧失。
> 当一个社会的子群体数量超过 7 个且彼此之间没有有效的调解机制时，
> 任何外部冲击都可能引发系统性崩溃。"
>
> — EP03: <标题>, 来源
```

**检查**：确保引用的是作者的原意，即使在改写也是"忠实转述"。

### I：Interpretation（解读）

**规则**：
- 用自己的话提炼方法论骨架
- 5-15 行（约 150-400 字）
- 聚焦于"这个技能的结构是什么"，而非"这个技能说了什么"
- 适合使用列表、流程图等结构化表达

**格式**：
```markdown
## I — Interpretation（解读）

这是一个用于评估文明/社会系统脆弱性的分析框架，其核心结构是：

1. **子群体计数**：统计社会内部具有独立利益诉求的子群体数量
2. **调解机制评估**：检查是否存在跨子群体的有效调解机制
3. **阈值判断**：当子群体数 > 7 且调解机制缺失 → 高风险
4. **冲击测试**：模拟一个外部冲击（战争/灾害/经济危机）的连锁反应

核心洞察：脆弱性不是由冲击的强度决定的，而是由系统内部的
连接结构决定的。这与工程学中的"冗余设计"概念相通。
```

### A1：Application by Author（作者应用）

**规则**：
- 描述作者如何在具体案例中应用这个框架
- 必须与视频内容对应
- 可以是 1 个详细案例或 2-3 个简要案例
- 长度：5-15 行

**格式**：
```markdown
## A1 — Application by Author（作者应用）

作者用这个框架分析了罗马帝国的衰落：

1. **子群体计数**：公元 3 世纪的罗马帝国内部有：元老院贵族、军队派系
   （至少 3 个）、行省总督、基督教团体、异教团体、奴隶群体 —— 超过 7 个。
2. **调解机制**：戴克里先的四帝共治试图建立调解机制，但反而加剧了分裂。
3. **阈值判断**：系统已超过阈值。
4. **冲击测试**：蛮族入侵（外部冲击）→ 各行省各自为战 → 西罗马帝国崩溃。

**来源时间戳**：[15:20 - 18:45]
```

### A2：Application Trigger（触发条件）

**规则**：
- 这是整个技能最关键的部分
- 定义**你**应该在什么情况下触发这个技能
- 使用条件式语言："当...时"
- 必须具体、可操作
- 长度：3-10 行

**格式**：
```markdown
## A2 — Application Trigger（触发条件）

当你遇到以下情况时，触发这个技能：

- **分析一个看似稳定的系统时**：任何组织（公司、国家、社区）
  如果在表面上看起来稳定，但内部利益群体不断增多
- **评估改革风险时**：当一个系统面临改革或外部冲击时，
  先用此框架评估系统脆弱性
- **投资/战略决策前**：在评估一个地区的长期稳定性时

**触发信号词**："分裂"、"崩溃"、"不稳定"、"内部分歧"
```

**设计原则**：A2 是整个技能的"API 接口"。一个好的 A2 让使用者在实际工作中自然联想到这个技能。

### E：Execution（执行步骤）

**规则**：
- 具体的、可操作的步骤
- 每一步以动词开头
- 包含"怎么做"而非"为什么做"
- 长度：5-10 步

**格式**：
```markdown
## E — Execution（执行步骤）

1. **列出所有利益群体**：画出系统地图，识别所有具有独立利益的子群体
2. **计数**：统计子群体数量，记录是否超过 7
3. **检查调解机制**：是否存在有效的跨群体沟通/协商机制？
   - 如有，评估机制的有效性（1-10 分）
   - 如无，标记为高风险项
4. **绘制影响关系图**：画出群体之间的依赖和冲突关系
5. **选择外部冲击**：选取 3 个可能的外部冲击情景
6. **模拟连锁反应**：对于每个冲击，追踪它在系统中的传播路径
7. **综合评估**：基于以上分析，输出脆弱性评级（高/中/低）
```

### B：Boundary（边界条件）

**规则**：
- 诚实说明技能的适用范围
- 至少包含 2 个"不使用"场景
- 包含技能的已知局限
- 长度：3-8 行

**格式**：
```markdown
## B — Boundary（边界条件）

**不适用场景**：
- 分析对象是**同质性高的小型群体**（如 10 人以下的创业团队）
- 系统处于**革命性变革期**而非渐进变化期（框架假设渐变）

**已知局限**：
- 阈值 7 来自经验观察，不同系统可能有不同阈值
- 框架无法预测"冲击"何时发生，只能评估系统对冲击的敏感度
- 调解机制的"有效性"评分依赖主观判断
```

## 三、质量检查表

每篇 SKILL.md 发布前必须通过以下检查：

### 必检项（必须全部通过）

| # | 检查项 | 标准 | 检查方法 |
|---|--------|------|---------|
| 1 | R 真实性 | 引用来自视频，≤150 字 | 检查时间戳，与原视频对照 |
| 2 | I 结构性 | 5-15 行，有自己的框架提炼 | 不是复述，而是结构再造 |
| 3 | A1 实例性 | 包含具体案例 | 能指出案例在视频中的位置 |
| 4 | A2 条件性 | 使用"当...时"句式 | 读给一个不懂技能的人，TA 能知道何时用 |
| 5 | E 可操作性 | 每一步以动词开头 | 检查步骤是否为可执行的动作 |
| 6 | B 诚实性 | 至少 2 个不适用场景 | 思考"这个技能在什么情况下会误导我" |

### 加分项（非必须但推荐）

| # | 加分项 | 说明 |
|---|--------|------|
| 1 | 跨域类比 | 在 I 部分加入其他领域的类比 |
| 2 | 反例 | 在 B 部分加入一个实际的反例 |
| 3 | Mermaid 图 | 在 E 部分加入流程图 |
| 4 | 关键词索引 | 在底部加入 3-5 个搜索关键词 |

## 四、输出规范

### 文件命名

```
skills/<系列简称>-<技能英文关键词>.md
```

例如：`skills/GT-civilization-fragility-analysis.md`

### Frontmatter

每篇 SKILL.md 必须包含完整的 YAML frontmatter：

```yaml
---
name: "文明脆弱性分析框架"
description: "通过子群体计数和调解机制评估评估社会系统脆弱性的方法"
source_series: "Global Trends"
source_episodes: ["EP03", "EP05", "EP08"]
tags: ["社会分析", "风险评估", "系统思维"]
related_skills: ["GT-power-transition-framework"]
created: YYYY-MM-DD
updated: YYYY-MM-DD
verified: true
v2_prediction_strength: "weak"  # strong / weak / framework_only
---
```

      │

## 六、检查清单

- [ ] 每个通过候选已创建对应的 SKILL.md

- [ ] I 字段：方法论骨架 5-15 行
- [ ] A1 字段：具体案例，有时间戳

## 七、常见陷阱

| 陷阱 | 后果 | 避免方法 |
|------|------|---------|
| R 字段过长或改写 | 失去保真性 | 严格 ≤150 字，直接引用 |
| I 字段复述而非解读 | 技能无增量价值 | 问自己"这个技能的结构是_____" |
| A2 模糊（"当遇到困难时"） | 使用者不记得触发 | 用"当[具体场景]时"句式 |
| E 字段解释而非执行 | 技能不可操作 | 每步以动词开头 |
| B 字段敷衍 | 使用者误用 | 想象最可能误用的场景 |
