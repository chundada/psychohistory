# Stage 3：跨系列链接（Cross-Series Zettelkasten Linking）

> **目标**：在技能之间建立可导航的依赖关系，形成知识网络而非孤立文件
> **输入**：`skills/` 目录中的所有 SKILL.md 文件
> **输出**：`INDEX.md`（含 Mermaid 依赖关系流程图）

---

## 一、为什么需要跨系列链接

单独看，每个技能都是一篇独立的 SKILL.md。但技能的真实价值在于它们如何组合使用：

- **技能 A** 提供分析框架，**技能 B** 提供执行步骤——A 是 B 的前提
- **技能 C** 和**技能 D** 对同一问题给出不同答案——使用者需要知道何时用哪个
- **技能 E** 来自系列 1，**技能 F** 来自系列 2，但两者可以组合成更强大的工具

没有链接的技能库只是一堆文件。有链接的技能库才是一个系统。

---

## 二、三种链接类型

### 2.1 depends-on（依赖关系）

**定义**：技能 A 的 E（执行步骤）中使用了技能 B 的框架。

**示例**：
- "文明脆弱性分析"（GT#03）的 E 步骤中使用的"子群体计数"方法，
  依赖于"社会网络映射"（GT#01）中的关系图绘制技能。

**格式**：
```yaml
depends-on:
  - skill: "GT-social-network-mapping"
    reason: "子群体计数需要先绘制完整的社会关系图"
    optional: false  # true 表示"推荐但不必须"
```

### 2.2 contrasts-with（对比关系）

**定义**：两个技能针对同一类问题给出不同的方法论。

**示例**：
- "文明脆弱性分析"（GT#03）基于**结构分析**评估脆弱性
- "精英周期理论"（GT#07）基于**精英行为**预测崩溃
- 两者对同一文明可能给出不同结论

**格式**：
```yaml
contrasts-with:
  - skill: "GT-elite-cycle-theory"
    difference: "脆弱性分析看结构，精英理论看人——前者适用于长期趋势，后者适用于短期波动"
    resolution: "先做脆弱性分析（长期框架），再用精英理论（短期信号）交叉验证"
```

### 2.3 composes-with（组合关系）

**定义**：两个技能可以组合成一个更大的工作流。

**示例**：
- "文明脆弱性分析" + "冲击情景规划" = 完整的风险评估工作流
- 前者识别系统弱点，后者设计应对策略

**格式**：
```yaml
composes-with:
  - skill: "GT-scenario-planning"
    workflow: "1. 用脆弱性分析识别系统弱点 → 2. 用场景规划设计应对方案"
    output: "完整风险评估报告"
```

## 三、执行步骤

### Step 1：为每个技能添加 related_skills

修改每个 SKILL.md 的 frontmatter，添加 `related_skills` 字段：

```yaml
related_skills:
  - "GT-social-network-mapping"    # depends-on
  - "GT-elite-cycle-theory"        # contrasts-with
  - "GT-scenario-planning"         # composes-with
```

### Step 2：构建 INDEX.md

INDEX.md 是整个系统的导航枢纽，包含以下部分：

#### 2a：技能目录（按主题分组）

```markdown
# PSYCHOHISTORY 技能索引

> 最后更新：YYYY-MM-DD
> 技能总数：N
> 来源系列：M

## 主题分组

### 一、文明分析

| 技能 | 来源 | 类型 | 链接数 |
|------|------|------|--------|
| 文明脆弱性分析 | GT#03 | 框架 | 3 |
| 社会网络映射 | GT#01 | 工具 | 2 |
| 精英周期理论 | GT#07 | 框架 | 2 |

### 二、地缘政治

| 技能 | 来源 | 类型 | 链接数 |
|------|------|------|--------|
| 权力转移判断 | GT#12 | 框架 | 4 |
| ... | ... | ... | ... |
```

#### 2b：Mermaid 依赖关系流程图

```mermaid
graph LR
    A[文明脆弱性分析] -->|depends-on| B[社会网络映射]
    C[精英周期理论] -->|contrasts-with| A
    A -->|composes-with| D[冲击情景规划]
    D -->|depends-on| E[概率判断框架]
    F[权力转移判断] -->|depends-on| A
    F -->|contrasts-with| C
```

#### 2c：Glossary（术语表）

记录跨技能使用的关键术语及其统一含义：

| 术语 | 定义 | 首次出现 | 相关技能 |
|------|------|---------|---------|
| 子群体 | 具有独立利益诉求的社会单元 | GT#03 | 文明脆弱性分析、社会网络映射 |
| 调解机制 | 跨群体沟通/协商的制度安排 | GT#03 | 文明脆弱性分析 |

#### 2d：使用指南（按目标场景）

为常见分析目标提供技能组合推荐：

```markdown

推荐组合：
1. 文明脆弱性分析（GT#03）→ 初步评估
2. 社会网络映射（GT#01）→ 绘制利益关系
3. 精英周期理论（GT#07）→ 交叉验证

### 场景 2：预测地缘政治冲突
...
```

## 四、链接质量标准

| 标准 | 说明 | 违规示例 |
|------|------|---------|
| **有理由** | 每个链接必须附带理由 | "文明脆弱性分析与权力转移判断相关"（无理由） |
| **有方向** | depends-on 是有向的 | A depends-on B ≠ B depends-on A |
| **不冗余** | 不创建意义相同的平行链接 | "A 相关 B" + "B 相关 A"（改为双向注释） |
| **可操作** | composes-with 应包含具体工作流 | "A 和 B 可以一起用"（缺少如何用） |

---

## 五、维护规则

1. **当新增技能时**：必须检查与所有已有技能的链接关系
2. **当删除技能时**：必须更新所有指向被删除技能的链接
3. **当修改技能时**：检查是否影响已有链接的理由
4. **定期清理**：每月检查一次，删除过时链接

## 七、常见陷阱

| 陷阱 | 后果 | 避免方法 |
|------|------|---------|
| 创建无理由的链接 | 链接没有信息增量 | 每条链接必须有 reason 字段 |
| 创建循环依赖 | 使用者在 A→B→A 中困惑 | 检查流程图是否有环 |
| 忘记更新 INDEX.md | INDEX.md 与实际技能脱节 | 每次技能变更后运行检查清单 |
| 链接过载 | 每个技能关联 10+ 技能，失去导航意义 | 每个 skills 最多 5 个链接 |
