# Stage 1.5：四重验证（Quadruple Verification）

> **目标**：在进入 RIA++ 构建之前，过滤掉不合格的候选技能
> **输入**：`candidates/` 目录中的所有候选技能文件
> **输出**：`verified.md`（验证通过的技能清单）+ `rejected/`（被拒绝的技能及原因）

---

## 一、为什么需要 Stage 1.5

在书籍蒸馏中，三重验证（V1-V3）在 RIA 之后进行。但在视频系列场景中，提前验证有三个好处：

1. **节省 RIA 构建成本**：不合格的候选技能不值得投入 RIA 格式
2. **避免系统冲突**：V4 确保新技能与已有技能不矛盾
3. **跨剧集一致性检查**：同一作者可能在系列中前后矛盾

### 与原版的三重验证对比

| 维度 | 书籍蒸馏（原版） | 视频系列蒸馏（本版） |
|------|-----------------|-------------------|
| V1 | 跨章节一致性 | **跨剧集一致性**（更强） |
| V2 | 预测能力 | 预测能力（相同） |
| V3 | 独特性 | 独特性（相同） |
| V4 | 无 | **新增：系统兼容性** |

## 三、V1：跨剧集一致性检查

### 目的
确保同一个技能在不同剧集中表述一致，没有矛盾。

### 执行方法

1. **按技能主题分组**：将所有候选技能按主题分组（例如"文明兴衰分析"、"权力转移判断"等）
2. **交叉引用**：对同一主题的不同候选，检查核心主张是否一致
3. **矛盾检测**：寻找以下矛盾类型：

| 矛盾类型 | 示例 | 处理方式 |
|---------|------|---------|
| **直接矛盾** | EP03 说"X 导致 Y"，EP05 说"X 不导致 Y" | 标记为 rejected，记录矛盾原因 |
| **范围矛盾** | EP03 说"X 适用于所有文明"，EP07 说"X 仅适用于西方文明" | 取更精确的范围表述 |
| **条件矛盾** | EP03 说"在条件下 A 用方法 B"，EP11 说"在条件下 A 用方法 C" | 检查是否同一条件，若是则 rejected |
| **弱不一致** | EP03 的案例与 EP08 的结论有张力 | 在技能中标注"争议点" |

### 通过标准
- 没有直接矛盾
- 范围矛盾已通过协商解决（取更精确版本）
- 弱不一致已记录在技能中

### 拒绝原因模板
```markdown
## 拒绝：<技能名称>

**来源**：EP<编号>-extractor-<编号>-<名称>.md
**拒绝关卡**：V1 - 跨剧集一致性
**矛盾内容**：
- EP<编号> 主张：<引用>
- EP<编号> 主张：<引用>
**矛盾类型**：直接矛盾 / 范围矛盾 / 条件矛盾
**拒绝理由**：<具体说明>
```

## 四、V2：预测能力检查

### 目的
真正有价值的技能应该能产生可验证的预测——不只是"解释过去"，而是能"指导未来"。

### 评估标准

| 等级 | 定义 | 示例 | 决策 |
|------|------|------|------|
| **强预测** | 技能能产生明确、可 falsifiable 的预测 | "如果一个文明的新增人口中有 60% 以上集中在城市，则该文明在 20 年内将出现社会动荡" | 通过 |
| **弱预测** | 技能能产生方向性的判断，但无法精确验证 | "人口过度集中的文明更可能经历社会不稳定" | 通过，标记为"弱预测" |
| **无预测** | 技能只是描述性框架，无法产生任何预测 | "文明由地理、经济、文化三个因素塑造" | 标记为"框架"，仅限 RIA 中的 I 部分 |
| **事后解释** | 技能只解释已发生的事件，无法指导未来 | "回顾来看，XX 文明衰落是因为..." | 如果可转化为预测则改造，否则 rejected |

### 通过标准
- 强预测或弱预测 → 通过
- 无预测 → 降级为"框架"，不单独成技能
- 事后解释 → 如果能转化为预测则进入改造，否则 rejected

### 测试方法
对于每个候选技能，问自己：**"如果我掌握了这个技能，我能预测什么？"**

如果答案是否定的，该候选技能至少需要改造才能通过 V2。

## 五、V3：独特性检查

### 目的
排除那些已经是常识、或者已被现有技能覆盖的候选。

### 评估标准

| 检查项 | 说明 | 通过条件 |
|--------|------|---------|
| **非常识** | 该技能不是显而易见的常识 | 至少让 80% 的人觉得"有启发" |
| **非表面** | 该技能不是"多努力就会成功"这类空话 | 包含具体、可操作的结构 |
| **非已有** | 该技能与 PSYCHOHISTORY 已有技能不重复 | 查看 INDEX.md 确认无重叠 |
| **非冗余** | 该技能不是同一系列其他技能的简单变体 | 与其他候选技能的区分度明显 |

### 通过标准
- 所有 4 项检查都通过 → 通过 V3
- 任何一项不通过 → rejected（记录原因）

## 六、V4：系统兼容性检查

### 目的
确保新技能与 PSYCHOHISTORY 系统中已有的所有技能兼容。

### 这是视频系列蒸馏中的新增维度

书是一个封闭系统，不存在"与已有技能冲突"的问题。但电影/心理史学技能库是持续增长的，新技能可能与旧技能矛盾。

### 执行方法

1. **加载 INDEX.md**（或 `skills/` 目录下的所有 SKILL.md）
2. **接口点检查**：对于每个新技能，检查以下 5 个接口点：

| 接口点 | 检查内容 | 示例冲突 |
|--------|---------|---------|
| 触发条件 | 新技能的 A2 是否与现有技能的 A2 冲突？ | 技能 A："当分析师问 XX 时"，技能 B："当分析师问 XX 时（但回答不同）" |
| 执行步骤 | 新技能的 E 是否否定现有技能的 E？ | 技能 A："第一步做 X"，技能 B："第一步做 Y，不要做 X" |
| 适用边界 | 新技能的 B 是否与现有技能的 B 矛盾？ | 技能 A："适用于所有文明"，技能 B："不适用于海洋文明" |
| 核心假设 | 新技能的前提假设是否与现有技能冲突？ | 技能 A："人性是理性的"，技能 B："人性是非理性的" |
| 输出冲突 | 两个技能应用于同一问题是否产生矛盾结论？ | 逻辑推演测试 |

### 冲突处理优先级

| 发现冲突 | 处理方式 |
|---------|---------|
| 新技能错误 | 修改新技能 |
| 旧技能过时 | 标记旧技能为"已废弃"，更新 INDEX.md |
| 两者都正确但视角不同 | 添加 contrasts-with 链接，在技能中说明不同适用场景 |
| 无法协调 | 两者都保留，在技能中注明"争议领域" |

### 通过标准
- 没有无法协调的冲突
- 所有冲突都已记录并分配了处理方式

## 九、常见陷阱

| 陷阱 | 后果 | 避免方法 |
|------|------|---------|
| V1 过度严格 | 丢掉有价值但表述略有差异的技能 | 区分"直接矛盾"和"表达方式不同" |
| V2 对预测要求过高 | 筛选掉好的框架性技能 | 允许"弱预测"通过，只需标记 |
| V3 自我审查 | 认为"这个太简单了"而拒绝 | 对比 INDEX.md 确认，不要主观判断 |
| V4 忽略 | 系统积累矛盾 | 每次新建技能前必须运行 V4 |
