# 博弈论模型提取器 (Game Theory Extractor)

## 用途 (Purpose)

从视频转录文本中提取**博弈论模型**相关的知识条目。识别主讲人使用或引用的博弈论框架，包括经典模型（囚徒困境、斗鸡博弈、信号博弈、承诺机制、升级模型、不对称博弈）、策略互动分析、纳什均衡及其他博弈均衡概念。

## 范围 (Scope)

### 属于此提取器的内容
- 经典博弈模型及其变体的显式或隐式引用（囚徒困境、斗鸡博弈、猎鹿博弈、最后通牒博弈、保证博弈、消耗战等）
- 策略概念：承诺、可信承诺、信号、信号成本、反信号、升级、模糊策略、边缘政策
- 均衡分析：纳什均衡、子博弈完美均衡、贝叶斯纳什均衡、帕累托最优
- 博弈结构分析：参与者数量、对手识别、支付结构、回合顺序、信息完全/不完全
- 合作与背叛的动态分析，重复博弈中的以牙还牙策略
- 不对称博弈：权力不对称、信息不对称、能力不对称对策略选择的影响

### 不属于此提取器的内容
- 纯军事战术描述（无策略互动结构）
- 经济学基本原理（无博弈元素的价格理论）
- 外交辞令陈述（无底层策略模型）
- 组织管理流程（非策略性互动）

## 识别信号 (Identification Signals)

在文本中寻找以下模式和关键词：

### 关键词与短语
```
"囚徒困境"、"斗鸡博弈"、"猎鹿困境"、"信令"、"发信号"、"信号成本"
"承诺"、"可信承诺"、"绑定"、"边缘政策"、"走钢丝"
"不对称"、"不对称博弈"、"信息不对称"
"均衡"、"纳什均衡"、"帕累托"、"帕累托最优"
"背叛"、"合作"、"以牙还牙"、"针锋相对"
"升级"、"螺旋"、"报复"、"互相升级"
"博弈"、"游戏"、"玩法"、"规则"、"局"
"策略"、"战略"、"最优反应"、"占优"
"威慑"、"可信威慑"、"确保摧毁"
"支付"、"收益"、"成本"、"回报"
"协调"、"协调问题"、"协调博弈"
"消耗"、"消耗战"、"长期消耗"
"第一次打击"、"先发制人"、"后发制人"
```

### 上下文模式
- 涉及两个或多个参与者在规则框架下做出相互依赖决策的场景
- 某人的最优选择依赖于他人选择的分析
- "如果A这样做，那么B就会那样做"的连锁推理链
- 对可信度、承诺能力、信号真实性的讨论
- 对升级风险、失控螺旋、安全困境的分析

## 输出格式 (Output Format)

每个提取条目使用以下 YAML 结构：

```yaml
- id: GAME_001
  title: "[博弈模型名称]"
  type: framework
  source_episode: "[视频标题/期号]"
  source_quote: "[原文引用，标注时间戳]"
  summary: |
    [200-300字的中文摘要，包含：博弈参与方、策略空间、支付结构、均衡结果、现实对应案例]
  tags:
    - game_theory
    - [具体模型名称，如 prisoner_dilemma / chicken_game / signaling_game]
    - [应用领域，如 geopolitics / economics / war]
  trigger_conditions:
    - [触发条件1：什么情境下这个模型被应用]
    - [触发条件2：模型的适用范围或局限]
    - [触发条件3：模型预测的关键变量]
```

## 质量检查清单 (Quality Checklist)

- [ ] 提取的条目是否清晰定义了博弈的参与者（至少两方）？
- [ ] 是否明确描述了各参与者的策略选择空间？
- [ ] 是否记录了支付/收益结构（各结果对应的代价或收益）？
- [ ] 是否识别了均衡状态（收敛点、稳定或不稳定）？
- [ ] 是否为该博弈模型提供了现实世界的历史或当代案例？

## 预期数量 (Expected Quantity)

每个系列（~30期视频）预期提取 **15–25 个** 博弈论模型条目，涵盖至少 5 种不同的博弈类型。
