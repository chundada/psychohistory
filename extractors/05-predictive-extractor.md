# 预测模型提取器 (Predictive Extractor) ★ 最高优先级

## 用途 (Purpose)

从视频转录文本中提取**预测模型**相关的知识条目。这是整个心理史学知识蒸馏系统中最关键的提取器。识别主讲人构建的"信号→演绎→结果"预测链条，包括条件预测、拐点识别、情景分析、历史类比推演等预测方法论。每一个提取条目都应当成为未来进行预测性推理的可复用框架。

## 范围 (Scope)

### 属于此提取器的内容（极度重要——请严格包含以下所有类别）

#### 1. 预测链条 (Prediction Chains)
- 明确的"如果X发生，那么Y将发生"的因果推论
- 多步骤逻辑链：A→B→C→D 的推演过程
- 被主讲人明确标注为"预测"的陈述
- 条件预测：只有在前提条件满足时才成立的预测
- 时间绑定的预测：明确标注时间范围的预测（5年内、10-20年、本世纪末等）

#### 2. 拐点识别 (Inflection Point Identification)
- 历史或当代转折点的明确标识
- "临界点"、"转折点"、"分水岭"、"拐点"类分析
- 从量变到质变的突变时刻
- 趋势加速或减速的识别
- 路径依赖锁定的关键时刻

#### 3. 情景分析 (Scenario Analysis)
- 多个可能未来路径的分岔分析
- 最好情况/最坏情况/最可能情况的分类
- 情景的触发条件和概率评估
- 不可逆的路径依赖选择

#### 4. 信号识别方法论 (Signal Identification)
- 主讲人使用的先行指标或预警信号
- 数据模式或历史模式的重复识别
- 异常值/反常现象作为预警
- 沉默/缺失的信号（应该发生但没有发生的事情）

#### 5. 历史类比推演 (Historical Analogy Projection)
- 将历史模式应用到当代的预测
- "这就像19XX年的X事件"类的结构性类比
- 类比的有效性条件和局限性讨论

#### 6. 概率性陈述 (Probabilistic Statements)
- "很可能"、"不太可能"、"确定无疑"等概率性判断
- 对不确定性的量化或分级
- 概率判断的推理依据

### 不属于此提取器的内容
- 已经发生的历史事件的回溯性描述（除非用于构建预测框架）
- 纯哲学性或价值观陈述（无预测成分）
- 对个人心理状态的猜测（非结构层面的预测）
- 无明确因果链的模糊论断

### ⭐ 跨提取器协作：经济金融信号

E5 和 E6（经济金融提取器）有大量重叠区域。当检索命中以下类型的陈述时，E5 和 E6 都应提取：

- **经济崩溃预测的时间线**：包含"X月后"、"Y年"等具体时间锚点
- **资产泡沫破裂的触发事件**：明确的条件因果链
- **债务危机的连锁反应路径**：系统性传染的阶段性描述
- **制裁效果的评估与预测**：带有"如果A继续，则B将发生"的结构
- **货币体系变革的长期预测**：去美元化、数字货币等演进路径

**协作规则**：如果一条内容同时属于 E5 和 E6，E5 提取框架链，E6 提取经济机理。两者在最终构造时汇聚。

### 关键上下文模式（优先级由高到低）

**第一优先级 —— 显式预测**
- 含有明确未来时间和预测内容的陈述
- 主讲人明确说"我预测……"

**第二优先级 —— 条件推演**
- "如果/假如/假设X，那么Y"
- "一旦A实现，B将不可避免"

**第三优先级 —— 类比推演**
- "X和Y世纪前的Z情况类似"
- "这就像是……将重演"

**第四优先级 —— 趋势外推**
- "按照这个速度/趋势……"
- "如果这个模式继续……"

## 输出格式 (Output Format)

每个提取条目使用以下 YAML 结构。这是最关键的一类提取，务必内容丰富、结构完整：

```yaml
- id: PRED_001
  title: "[预测命题名称]"
  type: framework
  source_episode: "[视频标题/期号]"
  source_quote: "[原文引用，标注时间戳]"
  summary: |
    [300-400字的中文摘要。必须包含：
    1. 预测的初始信号/条件
    2. 推演逻辑链（A→B→C→D的每一步）
    3. 预测的结果/终点
    4. 时间范围
    5. 预测的确定性/概率评估]
  tags:
    - prediction
    - [预测类型，如 conditional / scenario / analogy / trend_extrapolation]
    - [领域，如 geopolitics / war / economy / civilization / technology]
  prediction_chain:
    signal: "[触发预测的初始信号或条件]"
    steps:
      - step: 1
        description: "[逻辑推演的第一步]"
      - step: 2
        description: "[逻辑推演的第二步]"
      - step: 3
        description: "[逻辑推演的第三步]"
    outcome: "[预测的最终结果]"
  time_horizon: "[短期/中期/长期，或具体年数]"
  confidence: "[高/中/低，及主讲人的确定性表述]"
  trigger_conditions:
    - "[条件1：预测成立的前提]"
    - "[条件2：使预测可能失败的变量]"
    - "[条件3：需要监控的关键信号]"
```

## 质量检查清单 (Quality Checklist)

- [ ] 提取的条目是否包含一个**可证伪的预测**（即存在可能证明它错误的方式）？
- [ ] 是否清晰追溯了**从信号到结论的完整逻辑链**？
- [ ] 是否标注了**时间范围**（即使模糊如"长期"也好于没有标注）？
- [ ] 是否识别了**预测的依赖条件**（什么情况下预测会失效）？
- [ ] 是否区分了**确定性预测**与**概率性情景**？
- [ ] **特别检查**：这个预测是主讲人本人的推演，还是引用他人的观点？（请在 summary 中注明归属）

## 预期数量 (Expected Quantity)

每个系列（~30期视频）预期提取 **20–40 个** 预测模型条目。这是所有提取器中最高的预期数量，因为预测型内容是该频道的核心竞争力。宁可多提取一些边缘案例，也不要遗漏任何有预测框架价值的内容。

> **注意**: 当内容同时符合预测模型和其他提取器（如博弈论、地缘政治）时，**同时录入两个提取器**，并在各自的条目标注交叉引用。但是预测提取器永远优先确保完整度。
