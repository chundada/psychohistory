# ⚠️ 失败模式与思维陷阱提取器 — 已废弃

> **废弃于 v4.2**。失败模式防护不再作为独立提取器运行。
>
> **替代方案**：反例陷阱→改为在 RIA+ 构造时**内嵌检查**（"这个框架在什么条件下失效？" → 写在 B 段）。不再需要单独的提取阶段。
>
> **新提取器**：E6 已置换为「经济金融模式提取器」，见 `06-finance-economy-extractor.md`。

## 范围 (Scope)

### 属于此提取器的内容
- **归因错误**：基本归因错误、自利归因、团体归因偏差、将结果归因于阴谋而非结构
- **确认偏差与动机推理**：只寻找支持自己观点的证据、忽略矛盾的证据、认知失调
- **过度简化**：单一原因解释、二元对立思维、将复杂系统归因于单一因素
- **类比滥用**：不当的历史类比、错误的"历史重演"判断、类比的选择性使用
- **群体思维**：共识压力、信息茧房、团体极化、沉默螺旋
- **乐观/悲观偏差**：系统性高估或低估风险、计划谬误、灾难化思维
- **分析师陷阱**：过度专业化忽视全局、专业术语遮蔽常识、模型依赖
- **权力与认知**：权力导致认知扭曲、傲慢、自满、对警告信号的忽视
- **时间偏差**：短视主义、对长期趋势的忽视、时间贴现、急功近利
- **叙事谬误**：对随机事件强加因果关系、过度叙事化、事后诸葛亮
- **预测失败的具体案例**：历史上知名预测失败的案例分析，失败原因的归纳
- **未知的未知**：黑天鹅事件、未知领域、模型之外的风险

### 不属于此提取器的内容
- 没有结构分析价值的简单错误（如日期记错）
- 个人道德评判（"这个人很坏"而非"这个人的认知偏差是……"）
- 虚构作品中的失败（除非用于说明真实世界的认知陷阱）
- 纯粹的运气因素——没有可供学习的结构性教训
- 不涉及认知或分析框架的技术故障

### 上下文模式
- 主讲人明确指出某个判断或分析是错误的后，分析其犯错原因
- 对历史人物或国家策略的批评，重点在认知层面而非道德层面
- 指出"当时所有人都认为……结果却是……"的情境
- 对知名预测失败的复盘分析
- 系统性地列出某一类错误的多个历史案例
- 以"人们常犯的错误是……"或"常见的思维陷阱是……"开头

## 输出格式 (Output Format)

每个提取条目使用以下 YAML 结构。类型固定为 `counter-example`：

```yaml
- id: FAIL_001
  title: "[失败模式/思维陷阱名称]"
  type: counter-example
  source_episode: "[视频标题/期号]"
  source_quote: "[原文引用，标注时间戳]"
  summary: |
    [200-300字的中文摘要，包含：失败模式的核心机制、导致失败的认知过程、
     典型案例、可识别的预警信号、避免该陷阱的方法]
  tags:
    - failure_mode
    - [陷阱类别，如 attribution_error / confirmation_bias / analogy_abuse]
    - [应用领域，如 intelligence / strategy / prediction / analysis]
  symptoms:
    - "[识别该思维陷阱的征兆1]"
    - "[识别该思维陷阱的征兆2]"
    - "[识别该思维陷阱的征兆3]"
  preventive_measures:
    - "[避免此陷阱的方法1]"
    - "[避免此陷阱的方法2]"
```

## 质量检查清单 (Quality Checklist)

- [ ] 提取的条目分析的是**结构性/系统性**的失败原因，而非个人层面的偶然失误？
- [ ] 是否提供了至少一个**具体的历史或当代案例**来支撑这个模式？
- [ ] 是否识别了该思维陷阱的**可识别症状**（基于文本中给出的信号）？
- [ ] 是否记录了主讲人提出的**避免或缓解**该陷阱的方法？
- [ ] 是否标注了该失败模式的**适用边界**（在什么情境下这种偏差最容易出现）？

## 预期数量 (Expected Quantity)

每个系列（~30期视频）预期提取 **10–18 个** 失败模式与思维陷阱条目。
