# 宗教叙事结构提取器 (Religion Extractor)

## 用途 (Purpose)

从视频转录文本中提取**宗教叙事结构**相关的知识条目。识别主讲人分析的宗教历史模式：弥赛亚叙事、千禧年主义、一神教革命、神话结构、圣典与阐释体系、宗教的社会功能、宗教与文明的互动关系等。

## 范围 (Scope)

### 属于此提取器的内容
- 弥赛亚叙事结构：救世主降临、末世预言、审判与救赎、天选之民叙事
- 千禧年主义/千年王国运动：千禧年期盼、末世论、启示录式思维、革命性宗教运动
- 一神教分析：一神教革命、多神教到一神教的转型、唯一真神的政治含义
- 神话结构：创始神话、英雄之旅、牺牲与复活原型、二元论（光明vs黑暗）
- 宗教与社会结构：宗教与法律、政教关系、教士阶层、宗教改革运动
- 圣典阐释学：经文的解释传统、正统与异端的划定、诠释权的争夺
- 宗教的动力机制：宗教复兴、原教旨主义、世俗化理论、宗教市场论
- 宗教与文明互动：轴心时代、宗教对政治结构的塑造、宗教作为文明载体
- 仪式与社会整合：集体仪式、阈限阶段、涂尔干式的社会凝聚分析

### 不属于此提取器的内容
- 单纯的神学教义阐述（未涉及社会/历史分析）
- 个人灵性体验或信仰见证
- 宗教场所和建筑描述（无叙事结构分析）
- 纯宗教艺术史（图像学解读除外）
- 宗教冲突中的具体事件报道（未提炼模式）

### 上下文模式
- 分析宗教教义如何塑造信徒的行为模式和政治偏好
- 比较不同宗教的叙事结构相似性
- 讨论宗教在文明兴衰中的角色（既作为凝聚力也作为分裂源）
- 宗教运动与政治变革的关系分析
- 圣典的某段文本被用来解释后续的历史发展
- 当代事件被放在千年以上的宗教叙事传统中解读

## 输出格式 (Output Format)

每条目使用以下 YAML 结构。根据内容性质选择 `case`（案例性/事件性）或 `principle`（模式性/规律性）：

```yaml
- id: REL_001
  title: "[宗教叙事/模式名称]"
  type: case | principle
  source_episode: "[视频标题/期号]"
  source_quote: "[原文引用，标注时间戳]"
  summary: |
    [200-300字的中文摘要，包含：叙事/模式的核心结构、宗教传统归属、历史语境、
     社会影响机制、与政治/文明的互动关系]
  tags:
    - religion
    - [宗教传统，如 christianity / islam / buddhism / judaism / pagan]
    - [叙事类型，如 messianic / apocalyptic / monotheism / myth]
    - [分析维度，如 social_function / political_impact / narrative_structure]
  trigger_conditions:
    - [触发条件1：此叙事在何种社会条件下被激活/复兴]
    - [触发条件2：叙事对群体行为的影响方向]
    - [触发条件3：叙事的可持续性和自我限制因素]
```

## 质量检查清单 (Quality Checklist)

- [ ] 提取的内容是否超越了单一宗教的范畴（具有跨宗教比较或通用性）？
- [ ] 是否区分了该条目是具体历史案例（case）还是可重复模式（principle）？
- [ ] 是否记录了该宗教叙事如何影响社会行为或政治结构？
- [ ] 是否标注了叙事的起源时间和重大演变节点？
- [ ] 是否识别了该叙事的"竞争者"或"对立叙事"？

## 预期数量 (Expected Quantity)

每个系列（~30期视频）预期提取 **8–15 个** 宗教叙事结构条目。
